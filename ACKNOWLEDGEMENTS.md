## Clean-room influence statement

Conn's native shell was implemented clean-room from its own product plan using
public AppKit, SwiftUI, CoreGraphics, and Carbon APIs. Source code was not copied
from either reference project.

The studied ideas were limited to established native-window concepts:
accessory-app lifecycle, nonactivating panels, all-Spaces and full-screen window
behavior, top-center display geometry, and observing display changes. Conn uses
its own App Server architecture, state model, controls, accessibility behavior,
packaging, and visual implementation.
